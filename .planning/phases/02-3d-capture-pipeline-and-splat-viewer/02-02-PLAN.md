**Step 1: Install spark.js**
```bash
npm install @sparkjsdev/spark
```

**Step 2: Create `src/components/viewer/SplatScene.tsx`**
This is a `'use client'` component that:
- Imports `SplatMesh` and `SparkRenderer` from `@sparkjsdev/spark`
- Calls `extend({ SplatMesh, SparkRenderer })` to register as R3F JSX elements (per the official spark-react-r3f example pattern)
- Exports a `SplatScene` component that accepts `{ url: string }` prop
- Renders `<sparkSparkRenderer />` and `<sparkSplatMesh args={[{ url }]} />` inside the R3F scene
- Adds `<ambientLight intensity={0.5} />` for basic lighting

**Step 3: Create `src/components/viewer/GaussianSplatViewer.tsx`**
This is a `'use client'` component that:
- Renders an R3F `<Canvas>` filling 100% width/height with `gl={{ antialias: true }}`
- Sets default camera at `position: [0, 5, 5]` with `fov: 60`
- Includes `<SplatScene url={splatUrl} />` as child
- Includes `<OrbitControls />` from `@react-three/drei` for initial orbit interaction
- Props: `{ splatUrl: string }`
- Named export: `GaussianSplatViewer`

**Step 4: Create a test page at `src/app/tour/[slug]/splat-test/page.tsx`**
This is a `'use client'` page that:
- Uses `dynamic(() => import('@/components/viewer/GaussianSplatViewer'), { ssr: false })` to avoid SSR of WASM
- Shows a loading spinner while the dynamic import resolves
- Renders the GaussianSplatViewer with a hardcoded test SPZ URL. Use the official spark.js demo file: `https://sparkjsdev.github.io/spark/examples/guitar/bonsai.spz` (or any publicly available .spz file from the spark.js examples repo). If no public SPZ is available, use a local placeholder and note it in the summary.
- Includes a heading "Spark.js Compatibility Test" above the viewer
- The viewer fills the remaining viewport height

**Step 5: Update `next.config.ts` if needed**
If spark.js WASM loading fails, try these fixes in order:
1. Add `serverExternalPackages: ['@sparkjsdev/spark']` to next.config.ts experimental section
2. If still failing, copy the spark.js WASM file from `node_modules/@sparkjsdev/spark/dist/` to `public/wasm/` and configure the WASM path
3. Document which workaround was needed (or that none was needed) in the summary

**Important:** Follow the `dynamic(() => import(...), { ssr: false })` pattern used by PanoramaViewer.tsx. Never import spark.js at the top level of any server-renderable component.
