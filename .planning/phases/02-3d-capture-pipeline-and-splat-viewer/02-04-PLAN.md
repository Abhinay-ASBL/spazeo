  <name>Task 1: Create ReconstructionProgress component with multi-stage indicator and cancel</name>
  <files>src/components/tour/ReconstructionProgress.tsx</files>
  <action>
Create `src/components/tour/ReconstructionProgress.tsx` — `'use client'` component. Props: `{ tourId: Id<'tours'> }`.

**Data source:** `useQuery(api.reconstructionJobs.getActiveByTourId, { tourId })` — this returns the active job or null. The query auto-refreshes reactively when the job status changes (via webhook/cron updates).

**Multi-stage progress indicator:**
Display 6 stages as a horizontal stepper (or vertical on mobile):
1. Uploading — Upload icon
2. Queued — Clock icon
3. Extracting Frames — Film icon
4. Reconstructing — Cpu icon
5. Compressing — Archive icon
6. Complete — CheckCircle icon

Each stage shows:
- Icon (Lucide) — Gold (#D4A017) when active/complete, Muted (#6B6560) when pending
- Stage label
- Active stage has a pulsing animation (same Gold pulse ring concept from Phase 1 hotspots)
- Completed stages have a checkmark overlay

**Progress bar:** Below the stepper, a horizontal progress bar showing `job.progress`% with smooth transitions (`transition-all duration-500`). Show percentage text.

**Status text:** Below the bar, show contextual message:
- uploading: "Uploading your files..."
- queued: "Waiting for GPU availability..."
- extracting_frames: "Extracting frames from your video..."
- reconstructing: "Building 3D model... This usually takes 12-25 minutes"
- compressing: "Compressing model for web delivery..."

**Cancel button:** Below everything, a "Cancel Reconstruction" button (Ghost variant, text-red-400). On click, show a confirmation dialog (use Radix AlertDialog or a simple confirm modal): "Cancel reconstruction? GPU time already used will count toward your usage." On confirm, call `useMutation(api.reconstructionJobs.cancel)`.

**User can leave and return:** Add a note: "You can leave this page — we will notify you when complete" with a Bell icon.

**Failure state:** If `job.status === 'failed'`, show:
- Red error banner with AlertTriangle icon
- Human-readable error message from `job.error`
- Two buttons: "Retry" (re-triggers reconstruction with same inputs) and "Re-upload" (resets to upload UI)
- "Retry" calls `useMutation(api.reconstructionJobs.create)` with the same inputType and inputStorageIds from the failed job
- Failed jobs text: "This attempt does not count toward your monthly limit"

Style using the project design system: Surface background, appropriate text colors, Gold accents for active states, Error red for failure.
  </action>

  <done>ReconstructionProgress shows a live multi-stage stepper, progress bar, contextual status messages, cancel with confirmation, and failure state with Retry/Re-upload.</done>

  <name>Task 2: Create SplatPreview component and wire progress/preview into CaptureUpload</name>
  <files>src/components/tour/SplatPreview.tsx, src/components/tour/CaptureUpload.tsx</files>
  <action>
**SplatPreview.tsx** — `'use client'` component. Props: `{ jobId: Id<'reconstructionJobs'>, tourId: Id<'tours'> }`.

This component appears after reconstruction completes (status === 'completed').

**3D Preview:**
- Dynamic import the GaussianSplatViewer: `dynamic(() => import('@/components/viewer/GaussianSplatViewer'), { ssr: false })`
- Get the splat URL from the job's `outputStorageId` using a Convex query that resolves the storage URL
- Render the GaussianSplatViewer in a contained area (aspect-ratio 16/9 or fixed height ~400px) with orbit controls
- Show a loading spinner while the viewer loads

**Metadata display:**
Below the preview, show a grid of metadata cards:
- File size: format `outputMetadata.fileSizeBytes` as "XX.X MB"
- Gaussian count: format with comma separators (e.g., "523,847 gaussians")
- Processing time: format `outputMetadata.processingTimeMs` as "12m 34s"
- Input type: "Video Walkthrough" or "Multi-Angle Photos"

**Action buttons:**
- "Accept" button (Primary Gold variant, CheckCircle icon) — calls `useMutation(api.reconstructionJobs.acceptResult, { jobId })`. On success, toast "3D model accepted and linked to your tour!" The CaptureUpload component should then show the "3D Model active" status card.
- "Re-capture" button (Secondary/Ghost variant, RotateCcw icon) — resets the UI back to the upload state so the user can try again

**Wire into CaptureUpload.tsx:**
Update CaptureUpload to conditionally render based on the reconstruction job state:
1. If no active job and no splatStorageId on tour → show upload UI (existing)
2. If active job (uploading/queued/extracting/reconstructing/compressing) → show `<ReconstructionProgress tourId={tourId} />`
3. If job completed but not yet accepted → show `<SplatPreview jobId={jobId} tourId={tourId} />`
4. If tour has splatStorageId (previously accepted) → show "3D Model active" card with metadata and "Re-capture" option

Use `useQuery(api.reconstructionJobs.getByTourId, { tourId })` to determine current state.
