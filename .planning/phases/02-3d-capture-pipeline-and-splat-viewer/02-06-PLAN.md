**useSplatViewerStore.ts** — `'use client'` Zustand store:
```ts
type NavMode = 'dollhouse' | 'freeRoam' | 'hotspot'

interface SplatViewerState {
  navMode: NavMode
  transitioning: boolean
  controlsVisible: boolean   // auto-hide on mobile after 3s
  joystickVector: { x: number, y: number }  // for VirtualJoystick -> NavigationModes communication
  setNavMode: (mode: NavMode) => void
  setTransitioning: (v: boolean) => void
  setControlsVisible: (v: boolean) => void
  setJoystickVector: (v: { x: number, y: number }) => void
}
```
- Default navMode: 'dollhouse'
- setNavMode: only changes if not transitioning and mode is different

**NavigationModes.tsx** — `'use client'` R3F component (rendered inside Canvas):
This component manages camera behavior based on the active navMode. It reads `navMode` and `transitioning` from the Zustand store.

**Dollhouse mode:**
- Use a high perspective camera (position: [0, 12, 12], lookAt: [0, 0, 0]) to fake orthographic feel — per research, actual orthographic switching causes visual pops. Use high position + narrow FOV (~30) for a quasi-orthographic look.
- OrbitControls with rotation, zoom, and pan enabled. Max polar angle ~80deg to prevent going under the floor.

**Free-roam mode:**
- Camera at eye height (y=1.6m). PointerLockControls or custom look controls on desktop. Mouse drag to look around.
- Click-to-move: raycast from mouse click onto an invisible floor plane (y=0). On hit, smoothly lerp camera.position to the clicked point (keeping y=1.6m) over ~0.5s using useFrame.
- Camera movement is a smooth glide, not a teleport.

**Hotspot mode:**
- Camera behaves like free-roam but movement is restricted to hotspot-to-hotspot. Click a hotspot marker to fly to its position.
- OrbitControls with limited orbit (look around from current position) but no panning.

**Mode transitions (animated, ~1 second):**
- When navMode changes, set `transitioning: true` in store
- Animate camera from current position/rotation to the target mode's default position using useFrame with a lerp (factor ~0.05 per frame for smooth easing)
- After transition completes (distance to target < 0.01), set `transitioning: false`
- Cancel any in-progress animation before starting a new one (use a ref to track target)
- Disable mode switcher buttons during transitions

Use Three.js Vector3.lerp and Quaternion.slerp for smooth interpolation. Do NOT use gsap — keep it pure Three.js/R3F.

**VirtualJoystick.tsx** — `'use client'` DOM component (rendered outside Canvas):
- Only visible when navMode === 'freeRoam' AND on touch devices (detect via `'ontouchstart' in window` or media query `pointer: coarse`)
- Creates a nipplejs joystick manager in a container div positioned at bottom-left (left-4, bottom-20)
- On joystick move: calculate direction vector from joystick angle, store in Zustand via `setJoystickVector`
- On joystick end: reset vector to {0, 0}
- NavigationModes useFrame loop reads joystickVector and moves camera accordingly (translate along forward/right vectors at eye height)
- Semi-transparent joystick appearance: 120px zone size, 40px handle
- Destroy manager on unmount
