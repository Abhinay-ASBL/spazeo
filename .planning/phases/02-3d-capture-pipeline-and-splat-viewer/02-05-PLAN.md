Create `convex/reconstructionActions.ts` with a `submitToRunPod` action:

**Action: submitToRunPod**
- Type: `action` (not mutation — it calls external APIs)
- Args: `{ jobId: v.id('reconstructionJobs') }`
- Auth: Verify user identity
- Flow:
  1. Read the job record via `ctx.runQuery(internal.reconstructionJobs.getById, { jobId })` — add this internal query to reconstructionJobs.ts if it doesn't exist
  2. Get download URLs for all input storage files via `ctx.storage.getUrl(storageId)` for each inputStorageId
  3. Update job status to 'queued' via `ctx.runMutation(internal.reconstructionJobs.updateStatus, { jobId, status: 'queued', progress: 5 })`
  4. Submit to RunPod serverless:
     ```
     POST https://api.runpod.ai/v2/${RUNPOD_ENDPOINT_ID}/run
     Headers: Authorization: Bearer ${RUNPOD_API_KEY}
     Body: {
       input: {
         inputType: job.inputType,
         inputUrls: downloadUrls,
         outputFormat: 'spz',
         maxGaussians: 750000
       },
       webhook: `${CONVEX_SITE_URL}/gpu-callback?secret=${RUNPOD_WEBHOOK_SECRET}`
     }
     ```
  5. Store the returned `runpodJobId` via `ctx.runMutation(internal.reconstructionJobs.updateStatus, { jobId, status: 'queued', progress: 10, runpodJobId: response.id, startedAt: Date.now() })`
  6. On any fetch error, mark job as failed via `ctx.runMutation(internal.reconstructionJobs.fail, { ... })`

**Environment variables needed (set via `npx convex env set`):**
- `RUNPOD_API_KEY` — RunPod API key
- `RUNPOD_ENDPOINT_ID` — Serverless endpoint ID
- `RUNPOD_WEBHOOK_SECRET` — Shared secret for webhook verification
- `CONVEX_SITE_URL` — The Convex HTTP action base URL (e.g., `https://xxx.convex.site`)

**HTTP Callback Endpoint in `convex/http.ts`:**

Add a new route:
```
path: '/gpu-callback'
method: 'POST'
```

Handler logic:
1. Extract `secret` from URL query params: `new URL(request.url).searchParams.get('secret')`
2. Verify against `process.env.RUNPOD_WEBHOOK_SECRET` — return 401 if mismatch
3. Parse JSON body. RunPod sends: `{ id, status, output, executionTime, delayTime }`
4. Handle status cases:
   - `status === 'COMPLETED'` and `output.splatUrl` exists:
     - Fetch the SPZ file from `output.splatUrl` (temporary RunPod URL)
     - Store in Convex storage: `const storageId = await ctx.storage.store(splatBlob)`
     - Call `ctx.runMutation(internal.reconstructionJobs.complete, { runpodJobId: body.id, outputStorageId: storageId, outputMetadata: body.output.metadata })`
   - `status === 'FAILED'`:
     - Call `ctx.runMutation(internal.reconstructionJobs.fail, { runpodJobId: body.id, error: body.output?.error || 'Reconstruction failed on GPU worker' })`
   - `status === 'IN_PROGRESS'` (intermediate update):
     - Map RunPod progress to our stages: determine stage from `output.stage` field if present, or estimate from `executionTime`
     - Call `ctx.runMutation(internal.reconstructionJobs.updateStatus, { ... })` with mapped status and progress percentage
5. Return 200 OK

Also add CORS preflight route for `/gpu-callback` (OPTIONS handler) following the existing pattern in http.ts.

**Stale Job Polling Cron in `convex/crons.ts`:**

Add a cron job `pollStaleReconstructionJobs` that runs every 10 minutes.

Create the corresponding action in `convex/reconstructionActions.ts`:
- `pollStaleReconstructionJobs` — internal action
- Query all reconstruction jobs with status in ('queued', 'extracting_frames', 'reconstructing', 'compressing') where `startedAt` is older than 30 minutes ago
- For each stale job that has a `runpodJobId`:
  1. Call RunPod status endpoint: `GET https://api.runpod.ai/v2/${ENDPOINT_ID}/status/${runpodJobId}`
  2. If RunPod says COMPLETED — process same as webhook (download SPZ, store, complete)
  3. If RunPod says FAILED — mark as failed
  4. If RunPod says IN_QUEUE or IN_PROGRESS — update status/progress but don't mark as failed (still processing)
  5. If RunPod returns 404 or the job is unknown — mark as failed with "Job not found on GPU server"
- For stale jobs older than 2 hours regardless of RunPod status — mark as failed with "Reconstruction timed out"
- Log each action taken for debugging (console.log in Convex action)

Register the cron in `convex/crons.ts`:
```ts
crons.interval('poll stale reconstruction jobs', { minutes: 10 }, internal.reconstructionActions.pollStaleReconstructionJobs)
```
