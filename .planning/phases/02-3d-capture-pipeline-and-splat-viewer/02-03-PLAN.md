  <action>
**CaptureUpload.tsx** — `'use client'` component. Props: `{ tourId: Id<'tours'> }`.

Structure:
- Two tab buttons at top: "Video Walkthrough" and "Multi-Angle Photos" — styled as pill toggles using Gold active state per design system
- Quota display at top-right: "X of Y reconstructions remaining this month" — reads `useQuery(api.reconstructionJobs.getRemainingQuota)`
- Below tabs, a two-column layout (single column on mobile): upload zone on left, tips panel on right

**Video Tab:**
- Uses react-dropzone for file drop. Accept: `{ 'video/mp4': ['.mp4'], 'video/quicktime': ['.mov'] }`. Max size: 500MB (500 * 1024 * 1024).
- On valid drop: create a `<video>` element to extract thumbnail (capture first frame via canvas) and duration. Store in local state.
- Show preview card: video thumbnail, filename, file size (formatted: "124.5 MB"), duration ("2m 34s")
- Show clear error messages on validation failure: "File too large — maximum 500MB" or "Unsupported format — upload MP4 or MOV"
- "Remove" button to clear the selected video

**Photo Tab:**
- Uses react-dropzone for multi-file drop. Accept: `{ 'image/jpeg': ['.jpg', '.jpeg'], 'image/png': ['.png'] }`. No individual file size limit.
- Live counter: "X of 10-30 photos uploaded" with green checkmark icon when X >= 10
- Renders `<CapturePhotoGrid>` with the uploaded files
- Validation: minimum 10 photos required, maximum 30. Show warning if < 10: "Upload at least 10 photos for good reconstruction quality". Show error if > 30: "Maximum 30 photos — remove some before continuing."

**Start Reconstruction button:**
- Disabled until valid upload (video selected OR >= 10 photos)
- Disabled if quota exhausted (remaining === 0) — show "No reconstructions remaining this month" tooltip
- On click: (1) Upload files to Convex storage using 3-step pattern (generateUploadUrl -> POST -> collect storageIds), showing upload progress bar. (2) Call `useMutation(api.reconstructionJobs.create)` with tourId, inputType, inputStorageIds. (3) On success, show toast "Reconstruction started!" and the UI should transition to show progress (handled by Plan 05).
- Button text: "Start Reconstruction" with a Rocket or Play icon from Lucide

**CapturePhotoGrid.tsx** — `'use client'` component. Props: `{ files: File[], onRemove: (index: number) => void }`.
- Renders a responsive grid (3 cols on mobile, 4 on md, 5 on lg) of photo thumbnails
- Each thumbnail: object-cover, rounded-lg, with an X button overlay to remove
- Shows filename truncated below each thumbnail

**CaptureTips.tsx** — `'use client'` component. Props: `{ tab: 'video' | 'photos' }`.
- Collapsible panel (expanded by default on first visit — use localStorage flag `spazeo-capture-tips-seen`)
- Header: "Capture Tips" with ChevronDown toggle icon
- Video tips (with small Lucide icons per tip):
  - "Walk slowly and steadily" (Footprints icon)
  - "Cover all corners of the room" (Maximize icon)
  - "Keep good lighting — turn on all lights" (Sun icon)
  - "Avoid mirrors and windows" (EyeOff icon)
  - "Record 1-3 minutes per room" (Clock icon)
- Photo tips:
  - "Take 10-30 photos from different angles" (Camera icon)
  - "Overlap each shot by ~60%" (Layers icon)
  - "Cover all walls and corners" (Maximize icon)
  - "Include floor and ceiling edges" (ArrowUpDown icon)

Use the project design system: Surface (#12100E) background, Gold (#D4A017) accents for active tab, text colors from palette, 8px border radius on cards, DM Sans body text.
