  <name>Task 1: Add reconstructionJobs table and tours splat fields to Convex schema</name>
  <files>convex/schema.ts</files>
  <action>
Add the `reconstructionJobs` table to `convex/schema.ts` with these fields:
- `tourId: v.id('tours')` — links to the tour being reconstructed
- `userId: v.id('users')` — the user who triggered reconstruction
- `inputType: v.union(v.literal('video'), v.literal('photos'))` — which upload path was used
- `inputStorageIds: v.array(v.id('_storage'))` — 1 video storageId or N photo storageIds
- `status: v.union(v.literal('uploading'), v.literal('queued'), v.literal('extracting_frames'), v.literal('reconstructing'), v.literal('compressing'), v.literal('completed'), v.literal('failed'), v.literal('cancelled'))` — multi-stage progress
- `progress: v.number()` — 0-100 percentage
- `runpodJobId: v.optional(v.string())` — RunPod job reference for polling/webhook matching
- `outputStorageId: v.optional(v.id('_storage'))` — the .spz file in Convex storage
- `outputMetadata: v.optional(v.object({ fileSizeBytes: v.number(), gaussianCount: v.number(), processingTimeMs: v.number() }))` — metadata shown in quality review
- `error: v.optional(v.string())` — human-readable error message on failure
- `startedAt: v.optional(v.number())` — timestamp when GPU processing began
- `completedAt: v.optional(v.number())` — timestamp when job finished
- Indexes: `by_tourId` on `['tourId']`, `by_userId` on `['userId']`, `by_status` on `['status']`

Add to the existing `tours` table two new optional fields:
- `splatStorageId: v.optional(v.id('_storage'))` — the accepted .spz file
- `splatMetadata: v.optional(v.object({ fileSizeBytes: v.number(), gaussianCount: v.number(), processingTimeMs: v.number(), inputType: v.union(v.literal('video'), v.literal('photos')) }))` — metadata for display

All new fields are `v.optional()` for backward compatibility with existing tour documents.

<task type="auto">
  <name>Task 2: Create reconstructionJobs.ts CRUD functions and tours.linkSplat mutation</name>
  <files>convex/reconstructionJobs.ts, convex/tours.ts</files>
  <action>
Create `convex/reconstructionJobs.ts` with these functions:

**Mutations (all require auth via `ctx.auth.getUserIdentity()`):**
- `create` — args: `{ tourId, inputType, inputStorageIds }`. Validates: user owns the tour, checks reconstruction limits per plan (Free: 1/mo, Starter: 5/mo, Pro: 20/mo, Enterprise: unlimited). To check limits, count reconstructionJobs for this userId in the current calendar month where status is NOT 'failed' and NOT 'cancelled'. Creates job with status 'uploading', progress 0. Returns the new job `_id`.
- `updateStatus` — internal mutation. args: `{ jobId, status, progress, runpodJobId?, startedAt? }`. Updates status and progress fields. Used by actions and HTTP callback.
- `complete` — internal mutation. args: `{ runpodJobId, outputStorageId, outputMetadata }`. Finds job by `runpodJobId`, sets status to 'completed', progress to 100, stores outputStorageId and outputMetadata, sets completedAt. Also creates a notification in the `notifications` table for the user: `{ userId, type: 'system', title: '3D Reconstruction Complete', message: 'Your tour "[tourTitle]" has been reconstructed. Review and accept the result.', read: false }`.
- `fail` — internal mutation. args: `{ runpodJobId, error }`. Finds job by runpodJobId, sets status 'failed', stores error string.
- `cancel` — authenticated mutation. args: `{ jobId }`. Validates user owns the job. Sets status to 'cancelled'.
- `acceptResult` — authenticated mutation. args: `{ jobId }`. Validates user owns the job, job is 'completed'. Calls `ctx.db.patch` on the tour to set `splatStorageId` and `splatMetadata` from the job's output fields. This is the "Accept" action from the quality review step.

**Queries:**
- `getByTourId` — args: `{ tourId }`. Returns the most recent reconstruction job for this tour (order by `_creationTime` desc, take first). No auth required (used by public viewer to show reconstruction status).
- `getActiveByTourId` — args: `{ tourId }`. Returns the job if it's in an active state (not completed/failed/cancelled). Used by the editor to show progress UI.
- `getByUserId` — authenticated query. Returns all jobs for the current user, ordered by creation time desc.
- `getRemainingQuota` — authenticated query. Returns `{ used: number, limit: number, remaining: number }` for the current month based on user plan.

Use the `RECONSTRUCTION_LIMITS` constant: `{ free: 1, starter: 5, professional: 20, business: 20, enterprise: Infinity }`.

In `convex/tours.ts`, add a `linkSplat` internal mutation that accepts `{ tourId, splatStorageId, splatMetadata }` and patches the tour. This is called by `acceptResult` above. Keep it separate for reuse.

Follow existing patterns: use `getAuthUser` helper if it exists in the codebase, or use the `ctx.auth.getUserIdentity()` pattern from existing mutations.
  </action>
