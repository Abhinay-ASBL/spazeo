  <action>
Add four new tables to the schema after the Phase 3 furniture tables section:

1. **floorPlanProjects** -- multi-floor project grouping:
   - userId: v.id('users')
   - name: v.string()
   - tourId: v.optional(v.id('tours')) -- floor plans can exist independently or linked to a tour
   - floorCount: v.number()
   - status: v.union(v.literal('uploading'), v.literal('extracting'), v.literal('editing'), v.literal('completed'))
   - createdAt: v.number()
   - updatedAt: v.number()
   - Indexes: by_userId, by_tourId

2. **floorPlans** -- DO NOT modify the existing floorPlans table. Instead, add a NEW table called `floorPlanDetails` to avoid breaking existing tour floor plan functionality. Fields:
   - projectId: v.optional(v.id('floorPlanProjects'))
   - tourId: v.optional(v.id('tours'))
   - userId: v.id('users')
   - floorNumber: v.number()
   - label: v.optional(v.string()) -- e.g. "Ground Floor", "First Floor"
   - imageStorageId: v.id('_storage')
   - originalFileType: v.union(v.literal('pdf'), v.literal('image'), v.literal('sketch'))
   - extractionStatus: v.union(v.literal('pending'), v.literal('processing'), v.literal('completed'), v.literal('failed'))
   - extractionJobId: v.optional(v.id('floorPlanJobs'))
   - geometry: v.optional(v.object({
       walls: v.array(v.any()),
       rooms: v.array(v.any()),
       doors: v.optional(v.array(v.any())),
       windows: v.optional(v.array(v.any())),
       fixtures: v.optional(v.array(v.any())),
       dimensions: v.optional(v.any()),
     }))
   - scale: v.optional(v.number()) -- pixels per meter
   - currentVersion: v.optional(v.number())
   - createdAt: v.number()
   - updatedAt: v.number()
   - Indexes: by_projectId, by_tourId, by_userId

3. **floorPlanVersions** -- edit history:
   - floorPlanId: v.id('floorPlanDetails')
   - versionNumber: v.number()
   - geometry: v.any() -- same structure as floorPlanDetails.geometry
   - source: v.union(v.literal('ai'), v.literal('user'))
   - createdAt: v.number()
   - Index: by_floorPlanId

4. **floorPlanJobs** -- extraction job queue (separate from aiJobs because aiJobs requires tourId):
   - floorPlanId: v.id('floorPlanDetails')
   - userId: v.id('users')
   - status: v.union(v.literal('pending'), v.literal('processing'), v.literal('completed'), v.literal('failed'))
   - imageStorageId: v.id('_storage')
   - output: v.optional(v.any())
   - error: v.optional(v.string())
   - duration: v.optional(v.number())
   - createdAt: v.number()
   - Indexes: by_floorPlanId, by_userId, by_status

Also add `floorPlanExtractionsUsed` as v.optional(v.number()) to the users table for plan limit tracking (separate counter from aiCreditsUsed).

Add a comment header: `// --- Phase 4: Floor Plan Extraction ---` before the new tables.

  <action>
Create 5 Convex backend files following existing patterns from aiHelpers.ts and aiActions.ts:

**convex/floorPlanProjects.ts** -- CRUD mutations/queries:
- `create` mutation: auth check, insert with status='uploading', createdAt/updatedAt=Date.now()
- `getById` query: auth check, return project
- `listByUser` query: auth check, query by_userId index, return sorted by createdAt desc
- `update` mutation: auth check, patch name/status/floorCount/tourId, set updatedAt
- `remove` mutation: auth check, delete project and cascade-delete child floorPlanDetails

**convex/floorPlans.ts** (operates on floorPlanDetails table):
- `create` mutation: auth check, insert with extractionStatus='pending', createdAt/updatedAt
- `getById` query: auth check, return floor plan
- `listByProject` query: auth check, query by_projectId index, sort by floorNumber
- `updateGeometry` mutation: auth check, patch geometry + updatedAt, increment currentVersion, call saveVersion internally
- `saveVersion` internalMutation: insert into floorPlanVersions with versionNumber and source
- `resetToAiVersion` mutation: auth check, find version 1 (source='ai') from floorPlanVersions, restore geometry, create new version with source='user'

**convex/floorPlanVersions.ts** -- version queries:
- `listByFloorPlan` query: auth check, query by_floorPlanId index, sort by versionNumber desc
- `getVersion` query: auth check, get specific version by floorPlanId + versionNumber

**convex/floorPlanJobs.ts** -- job lifecycle:
- `create` internalMutation: insert job with status='pending', createdAt
- `updateStatus` internalMutation: patch status, optional error
- `complete` internalMutation: patch status='completed', output, duration
- `fail` internalMutation: patch status='failed', error, duration
- `getByFloorPlan` query: auth check, query by_floorPlanId index, return latest

**convex/floorPlanActions.ts** -- AI extraction:
- `extractFloorPlan` action:
  1. Check plan limits using floorPlanExtractionsUsed on users table:
     - Free: 3/month, Starter: 15/month, Pro: 50/month, Enterprise: unlimited
     - If over limit, throw error with upgrade message
  2. Create job via runMutation(internal.floorPlanJobs.create)
  3. Update floor plan's extractionJobId and extractionStatus='processing'
  4. Get image URL from storage via ctx.storage.getUrl(imageStorageId)
  5. Call DashScope qwen3.5-plus vision endpoint (DASHSCOPE_CHAT_URL) with:
     - System prompt: architectural floor plan analyzer
     - User message with image_url and EXTRACTION_PROMPT (from RESEARCH.md)
     - response_format: { type: 'json_object' } -- DO NOT use max_tokens (may truncate JSON)
  6. Parse JSON response, validate it has walls and rooms arrays
  7. Save output via runMutation(internal.floorPlanJobs.complete)
  8. Update floor plan geometry and extractionStatus='completed'
  9. Create version 1 with source='ai'
  10. Increment floorPlanExtractionsUsed on user
  11. Create notification (type='system') for extraction complete
  12. On error: mark job and floor plan as failed, do NOT increment usage counter

Use the same DashScope URL and getDashScopeKey pattern from aiActions.ts. Use the cast `const internal = _internal as any` pattern to break circular type references.

All auth checks follow the standard pattern: `const identity = await ctx.auth.getUserIdentity(); if (!identity) throw new Error('Not authenticated')`
