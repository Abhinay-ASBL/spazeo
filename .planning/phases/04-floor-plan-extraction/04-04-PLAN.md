**src/stores/floorPlanEditorStore.ts** -- 'use client' Zustand store:

State shape:
```typescript
interface FloorPlanEditorState {
  // Geometry data (canonical source of truth, all coordinates in meters)
  geometry: FloorPlanGeometry  // { walls, rooms, doors, windows, fixtures, dimensions }

  // Tool state
  activeTool: 'select' | 'wall' | 'room' | 'door' | 'window' | 'eraser'

  // Selection
  selectedElementId: string | null
  selectedElementType: 'wall' | 'room' | 'door' | 'window' | null

  // Drawing in progress
  drawingPoints: { x: number, y: number }[]  // For wall/room drawing

  // Undo history (same pattern as Phase 3 furniture: 50-item cap with shift-oldest eviction)
  undoStack: FloorPlanGeometry[]
  redoStack: FloorPlanGeometry[]

  // Viewport
  viewportScale: number
  viewportPosition: { x: number, y: number }
  stageSize: { width: number, height: number }

  // Dirty flag for save tracking
  isDirty: boolean

  // Actions
  setGeometry: (geometry: FloorPlanGeometry) => void
  setActiveTool: (tool: string) => void
  selectElement: (id: string | null, type: string | null) => void

  // Geometry mutations (all push to undo stack before mutating)
  addWall: (wall: Wall) => void
  updateWall: (wallId: string, updates: Partial<Wall>) => void
  deleteWall: (wallId: string) => void
  moveWallEndpoint: (wallId: string, endpoint: 'start' | 'end', position: { x: number, y: number }) => void

  addRoom: (room: Room) => void
  updateRoom: (roomId: string, updates: Partial<Room>) => void
  deleteRoom: (roomId: string) => void

  addDoor: (door: Door) => void
  addWindow: (window: Window) => void
  deleteDoor: (doorId: string) => void
  deleteWindow: (windowId: string) => void

  // Undo/redo
  undo: () => void
  redo: () => void

  // Viewport
  setViewportScale: (scale: number) => void
  setViewportPosition: (pos: { x: number, y: number }) => void
}
```

Use `create` from zustand. Undo stack capped at 50 with shift-oldest eviction (same as Phase 3 furniture pattern). Every geometry mutation pushes current geometry to undoStack and clears redoStack.

**DiagramCanvas.tsx** -- 'use client', dynamically imported with SSR disabled:
- Uses react-konva Stage, Layer, Line, Circle, Text, Rect, Group
- Reads geometry and tool state from useFloorPlanEditorStore
- Multiple Konva Layers for performance (per RESEARCH.md Pitfall 4):
  - Grid layer (listening: false): light dotted grid pattern for spatial reference
  - Walls layer: each wall rendered as a thick Line with draggable Circle endpoints
  - Rooms layer: each room rendered as a closed polygon (Line with closed=true) with semi-transparent fill and Text label at centroid
  - Doors/Windows layer: doors as small arc symbols on walls, windows as parallel lines on walls
  - Dimensions layer (listening: false): dimension annotation lines with arrow markers and measurement text
  - Selection layer: highlight overlay on selected element (Gold stroke, thicker line)

Zoom/Pan:
- onWheel handler: scale by 1.05 per scroll step, zoom toward pointer position (exact pattern from RESEARCH.md Pattern 3)
- Stage draggable when Select tool is active (drag-to-pan)
- Clamp scale between 0.1 and 5.0
- Update viewportScale and viewportPosition in store on zoom/pan

Tool interactions:
- **Select tool**: click element to select (update store), drag wall endpoints to move them (onDragEnd updates wall start/end), drag rooms to move entire polygon
- **Wall tool**: click to set start point (show preview line following cursor), click again to set end point (calls addWall). Wall gets auto-generated ID ('w' + incrementing number).
- **Room tool**: click to add polygon points (show preview polygon with dotted lines), double-click to close polygon (calls addRoom with auto-generated ID, auto-calculated area). Minimum 3 points.
- **Door tool**: click on a wall to place door at click position (calls addDoor with wallId, position, default width 0.9m)
- **Window tool**: click on a wall to place window at click position (calls addWindow with wallId, position, default width 1.2m)
- **Eraser tool**: click element to delete it from geometry (calls delete mutation on store)

Confidence highlighting:
- Walls with confidence='low': use amber (#FBBF24) stroke
- Rooms with confidence='low': use amber-tinted fill
- Konva Tooltip on hover for low-confidence elements: "This wall may need correction" (use Konva Text positioned near cursor on mouseover)

Coordinate system: all geometry in meters (canonical). The Konva Stage uses a scale factor (pixels per meter) calculated from the Stage dimensions and the geometry bounding box. All conversions happen in the canvas component, not the store.
