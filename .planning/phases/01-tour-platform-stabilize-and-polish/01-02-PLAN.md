    2. Add `useAction` import from 'convex/react'.
    3. Add `const verifyPassword = useAction(api.passwordUtils.verifyTourPassword)`.
    4. Add state: `const [passwordVerified, setPasswordVerified] = useState(false)` and `const [verifying, setVerifying] = useState(false)`.
    5. Update `handlePasswordSubmit` to be async:
       ```ts
       const handlePasswordSubmit = async () => {
         if (!password.trim() || verifying) return
         setVerifying(true)
         try {
           const ok = await verifyPassword({ slug, password: password.trim() })
           if (ok) {
             setPasswordVerified(true)
           } else {
             setShowPasswordError(true)
           }
         } catch {
           toast.error('Verification failed. Please try again.')
         } finally {
           setVerifying(false)
         }
       }
       ```

    7. Update the password gate condition: show the password gate when `tour.requiresPassword && !passwordVerified`.
    8. When `passwordVerified === true`, fetch the full tour data by using the existing `tourData` but with a re-fetch trigger — simplest approach: call `api.tours.getBySlugUnlocked` (a new query needed) or reuse `api.tours.getById` with the tourId. Actually since `getBySlug` returns `{ requiresPassword: true, scenes: [] }` for password-protected tours, and verification is now separate, after `passwordVerified = true` the viewer needs to load scenes. Add a new public query `getBySlugAuthenticated` in tours.ts that returns full scenes if called, OR use the existing `getById` query (but that requires tourId, not slug).

    Cleanest approach: Add a new query in convex/tours.ts:
    ```ts
    export const getBySlugWithScenes = query({
      args: { slug: v.string() },
      handler: async (ctx, args) => {
        // Same as getBySlug but skips the password gate — only used client-side after password is verified
        const tour = await ctx.db.query('tours').withIndex('by_slug', q => q.eq('slug', args.slug)).unique()
        if (!tour || tour.status !== 'published') return null
        // Load scenes
        const scenes = await ctx.db.query('scenes').withIndex('by_tourId', q => q.eq('tourId', tour._id)).collect()
        const scenesWithHotspots = await Promise.all(scenes.map(async scene => {
          const hotspots = await ctx.db.query('hotspots').withIndex('by_sceneId', q => q.eq('sceneId', scene._id)).collect()
          const imageUrl = scene.imageStorageId ? await ctx.storage.getUrl(scene.imageStorageId) : null
          return { ...scene, hotspots, imageUrl }
        }))
        return { ...tour, scenes: scenesWithHotspots.sort((a,b) => a.order - b.order), requiresPassword: false }
      },
    })
    ```
    Then in the viewer: `const unlockedTour = useQuery(api.tours.getBySlugWithScenes, passwordVerified ? { slug } : 'skip')`. The `tour` variable becomes `unlockedTour ?? tourData`.
