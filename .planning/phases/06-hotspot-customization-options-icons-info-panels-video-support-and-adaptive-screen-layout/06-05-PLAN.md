    **D. Update handleHotspotClick** to also handle media hotspots with video content. The current handler only processes navigation type. Extend it:
    ```typescript
    const handleHotspotClick = useCallback(
      (hotspot: { type: string; targetSceneId?: string; _id?: string; content?: string; videoUrl?: string; title?: string }) => {
        if (hotspot.type === 'navigation' && hotspot.targetSceneId) {
          setActiveSceneId(hotspot.targetSceneId)
          return
        }
        // Media type with video content — open video modal directly
        if (hotspot.type === 'media') {
          const videoSrc = (hotspot as any).videoUrl || hotspot.content
          if (videoSrc) {
            useViewerStore.getState().openVideoModal(videoSrc, hotspot.title)
            return
          }
        }
        // Info / link / media without video — open info panel
        if (hotspot._id) {
          setActiveHotspot(hotspot._id)
        }
      },
      [setActiveHotspot]
    )
    ```

    **E. In the JSX return, add AnimatePresence blocks** INSIDE the outermost container `div` (after the `<PanoramaViewer>` and other absolute-positioned elements, before closing `</div>`):

    ```tsx
    {/* ── Hotspot Info Panel ── */}
    <AnimatePresence>
      {activeHotspot && (
        <HotspotInfoPanel
          key={activeHotspot._id}
          hotspot={activeHotspot as any}
          onClose={() => setActiveHotspot(null)}
        />
      )}
    </AnimatePresence>

    {/* ── Video Modal ── */}
    <AnimatePresence>
      {videoModalUrl && (
        <HotspotVideoModal
          key={videoModalUrl}
          url={videoModalUrl}
          title={videoModalTitle}
          onClose={closeVideoModal}
        />
      )}
    </AnimatePresence>
    ```

    CRITICAL: Each child of `AnimatePresence` MUST have a `key` prop for exit animations to fire. `HotspotInfoPanel` uses `key={activeHotspot._id}` and `HotspotVideoModal` uses `key={videoModalUrl}`.
