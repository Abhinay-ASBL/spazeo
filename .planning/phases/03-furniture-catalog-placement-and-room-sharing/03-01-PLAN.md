**furnitureItems** table:
- name: v.string()
- category: v.union(v.literal('sofas'), v.literal('beds'), v.literal('tables'), v.literal('chairs'), v.literal('storage'), v.literal('decor'))
- style: v.union(v.literal('scandinavian'), v.literal('modern'), v.literal('luxury'), v.literal('industrial'))
- glbStorageId: v.id('_storage')
- thumbnailStorageId: v.optional(v.id('_storage'))
- dimensions: v.object({ width: v.number(), depth: v.number(), height: v.number() }) — meters
- priceUsd: v.number()
- amazonUrl: v.optional(v.string()) — FURN-02
- tags: v.optional(v.array(v.string()))
- Indexes: by_category ['category'], by_style ['style']
- Search index: search_name with searchField 'name', filterFields ['category', 'style']

**furnishedRooms** table:
- tourId: v.id('tours')
- userId: v.id('users')
- title: v.string()
- slug: v.string()
- createdAt: v.number()
- updatedAt: v.number()
- Indexes: by_tourId ['tourId'], by_slug ['slug'], by_userId ['userId']

**placedFurniture** table:
- furnishedRoomId: v.id('furnishedRooms')
- furnitureItemId: v.id('furnitureItems')
- instanceId: v.string() — client-generated UUID for local state matching
- position: v.object({ x: v.number(), y: v.number(), z: v.number() })
- rotation: v.object({ x: v.number(), y: v.number(), z: v.number() })
- scale: v.object({ x: v.number(), y: v.number(), z: v.number() })
- Index: by_furnishedRoomId ['furnishedRoomId']

2. Create convex/furnitureItems.ts with:
- `list` query: accepts optional category and style filters, returns all matching items with glbUrl resolved via ctx.storage.getUrl(). No auth required (catalog is public).
- `search` query: accepts searchTerm string + optional category/style filters, uses searchIndex 'search_name'. No auth required.
- `getById` query: accepts id, returns single item with resolved URLs. No auth required.
- `seed` mutation: accepts array of items (for initial catalog population), requires auth. Inserts items in batch. This is an admin/dev utility.
- `generateUploadUrl` mutation: wraps ctx.storage.generateUploadUrl() for GLB/thumbnail uploads. Requires auth.

3. Create convex/furnishedRooms.ts with:
- `create` mutation: accepts tourId, title. Generates slug from title + random suffix. Requires auth. Returns the new room ID.
- `getBySlug` query: accepts tourSlug and furnishedRoomSlug. Joins furnishedRooms with placedFurniture and furnitureItems to return the full room state with item metadata. No auth required (public share page).
- `getByTourId` query: accepts tourId, returns all furnished rooms for a tour. Requires auth.
- `savePlacements` mutation: accepts furnishedRoomId and array of placement objects ({ furnitureItemId, instanceId, position, rotation, scale }). Deletes existing placements for the room, then inserts all new ones. Requires auth. This is the "save" action.
- `deleteRoom` mutation: accepts furnishedRoomId, deletes room and all associated placements. Requires auth.

Define types:
```typescript
interface PlacedItem {
  instanceId: string          // crypto.randomUUID()
  furnitureItemId: string     // Convex ID as string
  position: [number, number, number]
  rotation: [number, number, number]
  scale: [number, number, number]
  name: string
  price: number
  amazonUrl?: string
  glbUrl: string
}

type UndoAction =
  | { type: 'place'; item: PlacedItem }
  | { type: 'remove'; item: PlacedItem }
  | { type: 'transform'; instanceId: string; prevPosition: [number,number,number]; prevRotation: [number,number,number]; prevScale: [number,number,number] }

type InteractionMode = 'navigate' | 'furnish'
type TransformMode = 'rotate' | 'scale'
```

Store state and actions:
- `mode: InteractionMode` — defaults to 'navigate'
- `setMode(mode)` — switches between navigate and furnish
- `ghostItemId: string | null` and `ghostGlbUrl: string | null` — the catalog item being placed
- `setGhostItem(id, url)` — sets ghost preview (clicking a catalog item)
- `clearGhost()` — clears ghost (after placement or cancel with Escape)
- `placedItems: PlacedItem[]` — all placed furniture in current session
- `addItem(item)` — adds to placedItems, pushes { type: 'place', item } to undoStack
- `removeItem(instanceId)` — removes from placedItems, pushes { type: 'remove', item } to undoStack
- `updateTransform(instanceId, position, rotation, scale)` — updates transform, pushes previous state as { type: 'transform', ... } to undoStack
- `selectedId: string | null` — currently selected placed item
- `setSelectedId(id)` — sets selection (null to deselect)
- `transformMode: TransformMode` — defaults to 'rotate'
- `setTransformMode(mode)` — switches between rotate and scale
- `undoStack: UndoAction[]` — capped at 50 entries
- `undo()` — pops last action and reverses it: place -> remove item, remove -> re-add item, transform -> restore previous transforms
- `totalCost` — derived getter computed as sum of placedItems prices
- `loadFromSaved(items: PlacedItem[])` — replaces placedItems array (for loading saved room state), clears undo stack
- `reset()` — clears all state back to defaults

Export the store as `useFurnitureStore` and export all types (PlacedItem, UndoAction, InteractionMode, TransformMode).

Cap undoStack at 50 entries — when pushing, if length >= 50, shift the oldest entry off.
